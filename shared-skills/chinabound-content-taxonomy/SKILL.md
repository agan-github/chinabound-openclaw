# chinabound-content-taxonomy

Purpose: Shared content types, destination taxonomy, and audience segmentation.

When to use:
- visible to ChinaBound agents that need shared operating rules.
- use it to align payload shape, editorial standards, risk gates, and naming conventions.

Core rules:
1. Prefer official and partner-approved sources over public chatter.
2. Preserve `taskId`, `assignmentId`, `packageId`, and source references across handoffs.
3. Use shared schemas in `shared/schemas/` before inventing local structures.
4. Escalate yellow/red tasks to `human-gatekeeper` and/or `compliance-review`.
5. Keep outputs concise, structured, and traceable.
